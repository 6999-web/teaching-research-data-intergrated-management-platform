# 手动评分页面加载问题修复

## 问题描述
手动评分页面无法加载，浏览器控制台显示错误：
```
Uncaught (in promise) TypeError: Cannot read properties of undefined (reading 'score')
```

## 问题原因
在模板中使用 `formData.scores[itemKey]` 时，如果该 `itemKey` 还没有被初始化，就会导致访问 undefined 对象的属性错误。

### 问题发生的时机
1. 组件挂载时，`selfEvaluationContent` 还是 null
2. 模板开始渲染，尝试访问 `formData.scores[itemKey].score`
3. 但此时 `formData.scores[itemKey]` 还未初始化，值为 undefined
4. 访问 `undefined.score` 导致错误

## 修复方案

### 1. 添加安全检查（v-if）
在评分区域添加 `v-if` 条件，确保只有在数据初始化后才渲染：

```vue
<!-- 修改前 -->
<div class="scoring-area">
  <el-input-number v-model="formData.scores[itemKey].score" />
</div>

<!-- 修改后 -->
<div v-if="formData.scores[itemKey]" class="scoring-area">
  <el-input-number v-model="formData.scores[itemKey].score" />
</div>
```

### 2. 优化初始化逻辑
确保在加载自评表内容后立即初始化表单数据：

```typescript
// 修改前
const loadSelfEvaluation = async () => {
  const response = await selfEvaluationApi.get(props.evaluationId)
  selfEvaluationContent.value = response.data.content
}

// 修改后
const loadSelfEvaluation = async () => {
  const response = await selfEvaluationApi.get(props.evaluationId)
  selfEvaluationContent.value = response.data.content
  
  // 立即初始化表单数据
  initializeFormData()
}
```

### 3. 简化初始化函数
移除不必要的检查，直接赋值以确保响应式：

```typescript
// 修改前
if (!formData.scores[itemKey]) {
  formData.scores[itemKey] = { score: 0, comment: '' }
}

// 修改后
formData.scores[itemKey] = { score: 0, comment: '' }
```

### 4. 移除重复初始化
在 `onMounted` 中移除重复的初始化调用：

```typescript
// 修改前
onMounted(async () => {
  await Promise.all([
    loadSelfEvaluation(),
    loadAttachments()
  ])
  initializeFormData()  // 重复调用
})

// 修改后
onMounted(async () => {
  await Promise.all([
    loadSelfEvaluation(),  // 内部已调用 initializeFormData()
    loadAttachments()
  ])
})
```

## 技术说明

### Vue 3 响应式系统
在 Vue 3 中，使用 `reactive()` 创建的对象是响应式的，但需要注意：
1. 直接赋值新属性会自动变成响应式
2. 使用 `v-if` 可以避免访问未定义的属性
3. 在模板中访问嵌套属性时要确保父对象存在

### 最佳实践
1. 在访问嵌套属性前添加安全检查
2. 在数据加载完成后立即初始化相关状态
3. 使用 `v-loading` 指令显示加载状态
4. 使用 `v-if` 条件渲染避免访问未定义的数据
