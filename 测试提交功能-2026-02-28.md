## 📋 当前提交流程

点击"提交表单"按钮时，系统会自动执行以下步骤：

### 步骤 1: 验证表单
- 检查所有必填字段是否已填写
- 验证自评分是否在有效范围内

### 步骤 2: 保存自评表数据
- 将表单数据保存到数据库
- 获取 evaluation_id

### 步骤 3: 上传附件（如果有）
- 自动上传所有内联附件
- 包括 16 个字段的附件：
  - 8 个常规教学工作字段
  - 4 个特色与亮点项目
  - 4 个负面清单项目

### 步骤 4: 提交到考评小组
- 锁定表单，不可再修改
- 将数据提交到考评小组端

## ✅ 提交功能已实现

当前代码已经实现了表单和附件一起提交的功能：

```typescript
const handleSubmit = async (submitData: any) => {
  try {
    // 1. 保存自评表
    const saveResponse = await selfEvaluationApi.save({
      teaching_office_id: authStore.teachingOfficeId,
      evaluation_year: new Date().getFullYear(),
      content: formData
    })
    
    const evaluationId = saveResponse.data.evaluation_id
    
    // 2. 上传附件（如果有）
    if (attachments && attachments.length > 0) {
      const uploadSuccess = await uploadAttachments(evaluationId)
      if (!uploadSuccess) {
        ElMessage.warning('附件上传失败，但表单已保存')
      }
    }
    
    // 3. 提交到考评小组
    await selfEvaluationApi.submit(evaluationId)
    
    ElMessage.success('提交成功！数据已上传到考评小组端')
  } catch (error) {
    ElMessage.error('提交失败，请重试')
  }
}
```

## 🐛 常见问题排查

### 问题 1: 点击提交没有反应

**可能原因**：
- 表单验证失败
- 必填字段未填写

**解决方法**：
1. 检查是否有红色错误提示
2. 确保所有必填字段都已填写
3. 检查浏览器控制台是否有错误信息

### 问题 2: 提交失败，显示 403 错误

**可能原因**：
- 未登录或登录过期
- 权限不足

**解决方法**：
1. 重新登录
2. 确认使用的是教研室账号（director1）

### 问题 3: 附件上传失败

**可能原因**：
- 文件过大（超过 50MB）
- 文件格式不支持
- 网络问题

**解决方法**：
1. 检查文件大小和格式
2. 重新上传附件
3. 如果附件上传失败，表单仍会保存，可以稍后单独上传

### 问题 4: 看到旧版本的页面

**可能原因**：
- 浏览器缓存

**解决方法**：
1. 按 `Ctrl + Shift + R` 硬刷新
2. 清除浏览器缓存
3. 关闭浏览器重新打开
