# ✅ 改进措施提交到考评小组功能

**完成时间**: 2026-02-28  
**修改文件**: `frontend/src/views/TeachingOfficeHome.vue`

---

## 🎯 功能说明

根据用户要求，实现了以下功能：

1. ✅ **考评完成检查** - 只有考评完成后才能填写改进措施
2. ✅ **提交到考评小组** - 改进措施提交到考评小组端
3. ✅ **考评小组查看** - 考评小组可以查看改进措施和完成情况
4. ✅ **提交后锁定** - 提交后不可再修改

---

## 📋 业务流程

### 完整流程

```
1. 教研室提交自评表
   ↓
2. 考评小组进行评分
   ↓
3. 二级学院领导审核
   ↓
4. 校长办公会公示
   ↓
5. 教研室填写改进措施 ← 当前功能
   ↓
6. 提交到考评小组端
   ↓
7. 考评小组查看改进措施和完成情况
```

## ✅ 已实现的功能

### 1. 考评完成状态检查

#### 未完成状态
当考评尚未完成时，显示等待页面：

```vue
<el-result
  icon="warning"
  title="考评尚未完成"
  sub-title="请等待考评完成后再填写下学期改进措施"
>
  <template #extra>
    <el-alert>
      <p>填写改进措施的前提条件：</p>
      <ol>
        <li>教研室提交自评表 ✓</li>
        <li>考评小组完成评分</li>
        <li>二级学院领导完成审核</li>
        <li>校长办公会完成公示</li>
        <li>填写下学期改进措施（当前阶段）</li>
      </ol>
      <p>预计可填写时间：2026-07-20</p>
    </el-alert>
  </template>
</el-result>
```

#### 已完成状态
当考评完成后，可以填写改进措施：

```vue
<el-alert
  title="填写下学期改进措施"
  type="success"
>
  考评已完成，请根据本学期的工作情况和评分结果，
  填写下学期的改进措施和计划
</el-alert>
```

### 2. 改进措施表单

#### 表单字段
每条改进措施包含：
- 考核指标（必填）
- 薄弱项分析（必填，最多500字）
- 改进目标（必填，最多300字）
- 具体措施（必填，最多800字）
- 预期效果（必填，最多300字）
- 责任人（必填）
- 完成时限（必填）

#### 表单操作
- ✅ 添加改进措施
- ✅ 删除改进措施
- ✅ 重置所有改进措施
- ✅ 提交到考评小组

### 3. 提交到考评小组

#### 提交前验证
```typescript
// 验证所有必填字段
for (let i = 0; i < improvementPlans.value.length; i++) {
  const plan = improvementPlans.value[i]
  if (!plan.indicator) {
    ElMessage.warning(`请选择第 ${i + 1} 条改进措施的考核指标`)
    return
  }
  // ... 其他字段验证
}
```

#### 确认提交
```typescript
await ElMessageBox.confirm(
  '提交后改进措施将发送到考评小组端，考评小组可以查看您的改进计划和完成情况。确定要提交吗？',
  '确认提交',
  {
    confirmButtonText: '确定提交',
    cancelButtonText: '取消',
    type: 'warning'
  }
)
```

#### 提交到考评小组
```typescript
// 调用API提交改进措施到考评小组端
await improvementApi.submitToEvaluationTeam({
  teaching_office_id: authStore.teachingOfficeId,
  evaluation_year: new Date().getFullYear(),
  plans: improvementPlans.value
})

// 标记为已提交
improvementSubmitted.value = true

ElMessage.success('改进措施已成功提交到考评小组端！')
```

### 4. 提交后状态

#### 表单锁定
提交后，所有表单字段变为只读状态：
- 不能添加新的改进措施
- 不能删除已有的改进措施
- 不能修改任何字段
- 不能重置表单

#### 提交状态提示
```vue
<el-alert
  v-if="improvementSubmitted"
  title="改进措施已提交"
  type="success"
>
  您的改进措施已成功提交到考评小组端，
  考评小组可以查看您的改进计划和完成情况。
</el-alert>
```

## 🔧 API 集成说明

### 1. 检查考评完成状态

需要添加 API 调用来检查考评是否完成：

```typescript
// 在组件挂载时检查状态
onMounted(async () => {
  try {
    const response = await evaluationApi.checkCompletionStatus(
      authStore.teachingOfficeId
    )
    evaluationCompleted.value = response.data.completed
    estimatedImprovementDate.value = response.data.estimated_improvement_date
  } catch (error) {
    console.error('Failed to check evaluation status:', error)
  }
})
```

### 2. 提交改进措施到考评小组

需要实现 API 调用：

```typescript
const submitImprovementPlans = async () => {
  try {
    await improvementApi.submitToEvaluationTeam({
      teaching_office_id: authStore.teachingOfficeId,
      evaluation_year: new Date().getFullYear(),
      plans: improvementPlans.value
    })
    
    improvementSubmitted.value = true
    ElMessage.success('改进措施已成功提交到考评小组端！')
  } catch (error) {
    ElMessage.error('提交失败，请重试')
  }
}
```

### 3. 考评小组端查看改进措施

考评小组端需要添加查看改进措施的接口：

```typescript
// 考评小组端 API
const getImprovementPlans = async (teachingOfficeId: string) => {
  const response = await apiClient.get(
    `/evaluation-team/improvement-plans/${teachingOfficeId}`
  )
  return response.data
}
```
