# 快速测试 - 考评小组接收考评表

## 🎯 测试目标
验证考评小组端能否接收教研室端提交的考评表

## ⚡ 快速测试步骤

### 第一步：教研室提交考评表

1. **打开浏览器**
   ```
   访问：http://localhost:3000
   ```

2. **登录教研室账号**
   ```
   用户名：director1
   密码：password123
   ```

3. **填写自评表**
   - 点击左侧菜单"填写自评表"
   - 填写必填项（至少填写一些内容）
   - 滚动到底部

4. **提交表单**
   - 点击"提交表单"按钮
   - 等待提交成功提示
   - **记录提交时间**

5. **验证提交成功**
   - [ ] 看到"提交成功！数据已上传到考评小组端"提示
   - [ ] 没有错误信息

### 第二步：考评小组查看考评表

1. **退出当前账号**
   - 点击左下角"退出登录"

2. **登录考评小组账号**
   ```
   用户名：evaluator1
   密码：password123
   选择角色：评教小组
   ```

3. **进入手动评分页面**
   - 点击左侧菜单"手动评分"
   - 查看待评分列表

4. **验证能看到考评表**
   - [ ] 列表中显示"计算机科学教研室"
   - [ ] 提交时间格式正确：`2026-02-28 14:30:45`
   - [ ] 状态显示为"已提交"或"AI已评分"

5. **查看考评表详情**
   - 点击"开始评分"按钮
   - 验证：
     - [ ] 页面正常加载
     - [ ] 显示自评表内容
     - [ ] 显示附件（如果有）
     - [ ] 可以输入评分

## ✅ 成功标准

### 教研室端
- ✓ 能够登录
- ✓ 能够填写自评表
- ✓ 能够提交表单
- ✓ 提交后显示成功提示

### 考评小组端
- ✓ 能够登录
- ✓ 能够看到待评分列表
- ✓ 列表中显示教研室提交的考评表

## ❌ 如果测试失败

### 问题1：教研室提交失败
**症状**：点击"提交表单"后显示错误

**检查**：
1. 打开浏览器控制台（F12）
2. 查看 Console 标签的错误信息
3. 查看 Network 标签的请求状态

**可能原因**：
- 后端服务未运行
- 网络连接问题
- 权限不足

### 问题2：考评小组看不到列表
**症状**：手动评分页面显示"暂无待评分的教研室"

**检查**：
1. 确认教研室已经提交（第一步完成）
2. 打开浏览器控制台（F12）
3. 查看 Network 标签
4. 找到 `/api/scoring/evaluations-for-scoring` 请求
5. 查看响应数据

**可能原因**：
- 教研室还没有提交
- 考评表状态不正确
- API 返回数据为空

### 问题3：时间格式不对
**症状**：时间显示为 `2026/2/28 下午2:30`

**解决方法**：
1. 清除浏览器缓存（Ctrl + Shift + Delete）
2. 强制刷新页面（Ctrl + F5）
3. 重新登录

### 查看关键 API 请求

#### 教研室提交
```
POST /api/teaching-office/self-evaluation/{id}/submit
状态码：200
响应：{"evaluation_id": "...", "status": "submitted", ...}
```

#### 考评小组获取列表
```
GET /api/scoring/evaluations-for-scoring
状态码：200
响应：[{"id": "...", "teaching_office_name": "...", ...}]
```

在 Kiro 中查看后端进程输出，查找：
- 登录请求
- 提交请求
- 获取列表请求
- 错误信息
