# 教研室工作考评系统开发需求文档（对接kiro）

# 一、需求概述

开发一套教研室工作考评全流程数字化系统，覆盖教研室自评、AI自动评分、管理端审核评议、校长办公会审定、结果公示与分发全环节，实现考评流程标准化、数据流转自动化、审核追溯可查，提升考评效率与透明度，严格按照以下细节逻辑落地开发。

# 二、核心角色与对应权限

- 教研室端：登录系统填写自评表、填写教改项目/荣誉表彰个数、上传支撑附件、手动触发AI评分，查看最终考评结果（得分、细则、系统总结）。

- 管理端（含考评小组、考评办公室）：考评小组手动二次评分（权重高）、考评办公室复核评分、处理异常数据、手动发起公示、汇总评分结果、上传数据至校长办公会端，查看所有评审人打分记录。

- 校长办公会端：查看实时考评数据（图表+明细）、查看各环节分数变化、查看教研室横向排名（按不同指标排序）、一键审定考评结果。

# 三、全流程开发需求（核心重点）

## 3.1 教研室端功能（前端+交互）

1. 表单填写：在线编辑教研室工作考核表，支持填写各项自评内容，单独模块填写“教学改革项目个数”“荣誉表彰个数”，表单填写完成后可保存、预览。

2. 附件上传：点击表单最后一栏“附件”，跳转至独立附件上传页面（页面含表格，明确附件对应指标），支持多文件上传，支持上传各类证书、项目文件。

3. AI评分触发：表单与附件全部提交后，显示“触发AI评分”按钮，仅教师手动点击后，启动AI自动评分流程，未点击不触发。

4. 结果查看：公示完成后，可查看最终得分、详细评分细则、系统自动生成的综合感悟总结。

5. 限制：表单与附件提交后自动锁定，不可修改，直至管理端审核完成或打回补充。

## 3.2 AI自动评分模块（核心开发点）

1. 触发条件：仅教研室端教师手动点击“触发AI评分”按钮后启动，调用deepseekAPI。

2. 评分逻辑：同时解析自评表文字内容 + 附件支撑材料，两者结合完成评分。

3. 重点指标计分：教学改革项目、荣誉表彰，按“填写个数+附件解析个数”对照，结合预设分值标准计分。

4. 异常数据处理：
        

    - 当自评表填写个数（教改/荣誉）与附件解析个数不一致时，系统自动标记异常。

    - 异常提示：生成清晰对比说明（例：“自评表填2项国家级教改项目，附件仅上传1份证书”），同步至管理端，直接转人工复核，不自动通过审核。

5. 附件处理：AI自动按考核指标（如“教学改革项目”“荣誉表彰”）对附件分类归档，支持管理端后续调整分类标签。

## 3.3 管理端功能（前端+后端交互）

1. 评分功能：完全独立于AI评分，进行手动二次评分；保留每一位评审人的原始打分记录，不可删除、不可修改。

2. 权重设置：考评小组评分权重高于考评办公室，前端界面需突出显示考评小组的评分记录（便于追溯）。

3. 最终得分计算：由考评办公室综合所有评审人（考评小组+考评办公室）打分记录，录入管理端最终得分，系统保留该最终得分与对应汇总说明。

4. 异常处理：接收AI标记的异常数据，可查看详细对比说明，可选择打回教研室补充材料，或在审核时直接修正异常数据。

5. 数据同步：管理端完成二次评分、确定最终得分后，显示“上传至校长办公会”按钮，点击按钮后，通过HTTPS协议将所有考评数据（含评分记录、附件、异常处理结果）同步至校长办公会端。

6. 公示操作：校长办公会审定同意后，由考评办公室手动操作“发起公示”按钮，启动结果公示流程。

## 3.4 校长办公会端功能（前端+数据展示）

1. 数据接收：接收管理端通过HTTPS协议上传的所有考评数据，确保数据完整（无遗漏评分记录、附件）。

2. 实时监控：以图表、数据看板形式展示考评结果，包含：
       

    - 各教研室各环节历史分数变化（AI评分、管理端各评审人评分、管理端最终得分）；

    - 各教研室横向对比排名，支持按不同考核指标（如“教学过程管理”“课程建设”）单独排序、切换查看。

3. 结果审定：显示“审定”按钮，点击后可选择“同意公示”或“驳回重新审核”，同意后同步至管理端，允许考评办公室发起公示；驳回后需反馈驳回原因至管理端。

## 3.5 公示与结果分发（后端+前端）

1. 公示触发：仅管理端考评办公室手动点击“发起公示”按钮后，启动公示（无需系统自动发起）。

2. 结果分发：公示无异议后（系统无需处理异议流程，默认公示后无异议），系统自动将最终考评结果分发至管理端与教研室端：

    - 教研室端：显示最终得分、详细评分细则（含所有评审人打分记录）、系统基于综合指标自动生成的感悟总结（例：“该教研室课程建设指标得分突出，但教学改革项目数量有待提升”）；

    - 管理端：显示所有教研室最终得分、审定结果，便于归档。

3. 感悟生成：系统基于各教研室综合考核指标完成情况（得分高低、指标达标情况）自动生成总结，无需人工填写。

## 3.6 数据追溯与归档（后端）

1. 全流程日志：保留所有操作日志（表单提交、AI评分、手动评分、异常处理、数据同步、审定、公示），可追溯每一步操作人、操作时间。

2. 附件归档：AI分类后的附件，支持管理端调整分类标签，所有附件与对应教研室、考核指标关联，长期归档，可随时查阅、下载。

3. 评分追溯：所有评审人打分记录、AI评分记录、管理端最终得分，均永久保留，不可篡改，便于后续审计、追溯。

# 四、开发核心要求

1. 接口要求：AI评分需调用deepseekAPI，数据同步（管理端→校长办公会端）需采用HTTPS协议，确保数据传输安全。

2. 交互要求：所有按钮点击后需有明确反馈（加载提示、成功/失败提示），表单填写支持实时校验（必填项、格式校验）。

3. 异常处理：仅支持AI标记的数据异常，异常提示需清晰、具体，便于管理端快速处理；无其他异常场景（如登录异常、文件上传失败），暂不考虑。

4. 展示要求：校长办公会端的图表、数据看板需直观，支持数据切换、筛选；管理端需突出显示考评小组评分记录。
