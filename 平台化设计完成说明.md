**更新时间**: 2026-02-06

## 设计理念

从"网站"转变为"平台"，采用左侧导航栏 + 右侧内容区的经典平台布局。

## 主要改进

### 1. 布局结构 📐

#### 之前（网站风格）
- ❌ 顶部导航栏
- ❌ 英雄区域（Hero Section）
- ❌ 垂直滚动的长页面
- ❌ 所有功能堆叠在一起

#### 现在（平台风格）
- ✅ 左侧固定导航栏
- ✅ 右侧内容区域
- ✅ 顶部标签页切换
- ✅ 功能分类清晰

### 2. 左侧导航栏 🎯

**结构**:
```
┌─────────────────┐
│   校徽 + 标题    │
├─────────────────┤
│   教研室端       │
```

**特点**:
- 深蓝色背景 (#1e3a5f)
- 固定定位，始终可见
- 三个端口清晰分类
- 底部显示用户信息

### 3. 右侧内容区 📊

## 交互设计

### 导航交互
1. 点击左侧端口 → 切换到对应端口
2. 自动显示该端口的标签页
3. 默认选中第一个标签页
4. 显示对应的功能列表

### 卡片交互
- **悬停效果**: 
  - 边框变蓝色
  - 向上浮动 4px
  - 显示箭头图标
  - 阴影加深

### 响应式设计
- **桌面**: 完整显示
- **平板**: 功能卡片单列
- **手机**: 侧边栏收缩，只显示图标

## 技术实现

### Vue 3 Composition API
```typescript
// 响应式数据
const activeModule = ref(0)  // 当前选中的端口
const activeTab = ref(0)     // 当前选中的标签页

// 计算属性
const currentTabs = computed(() => moduleTabs.value[activeModule.value])
const currentFunctions = computed(() => moduleFunctions.value[activeModule.value][activeTab.value])
```

### 布局技术
- **Flexbox**: 侧边栏和主内容区布局
- **Grid**: 功能卡片网格布局
- **Fixed**: 侧边栏固定定位
- **Sticky**: 标签栏粘性定位

- 侧边栏宽度: 200px

### 平板 (768px - 1024px)
- 侧边栏宽度: 200px
- 功能卡片: 单列

## 性能优化

### 1. 计算属性缓存
- 使用 `computed` 缓存计算结果
- 避免重复计算

### 2. 条件渲染
- 只渲染当前选中的内容
- 减少 DOM 节点

### 3. CSS 动画
- 使用 CSS transition
- 硬件加速
