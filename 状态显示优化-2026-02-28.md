```

  ↓

  const types: Record<string, string> = {
    'draft': 'info',

    'finalized': 'success',       // 改为 success (绿色)
    'published': 'success'
  }
  return types[status] || 'info'
}
```

## 优化理由

### 1. 简化标签文本
- "已锁定" → "已提交"：更直观，用户不需要理解"锁定"的技术含义
- "AI已评分" → "待评分"：强调需要人工评分的状态
- "已手动评分" → "已评分"：简化表述，更清晰

### 2. 优化颜色逻辑
- **橙色 (warning)**：需要处理的状态（已提交、待评分）
- **绿色 (success)**：已完成的状态（已评分、已确定最终得分、已公示）
- **蓝色 (primary)**：中间状态（待最终确定）
- **灰色 (info)**：初始状态（草稿）

### 3. 提升用户体验
- 颜色更有逻辑性，一眼就能看出哪些需要处理
- 标签文本更简洁，易于理解
- 状态流转更清晰
