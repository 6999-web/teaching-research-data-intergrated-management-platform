# 附件上传下载问题诊断和修复

## 问题描述
教研室端上传的附件，在考评小组端的手动评分页面无法显示和下载。

## 根本原因分析

### 1. API端点路径正确性 ✅
- 后端API路径：`/api/teaching-office/attachments/{evaluation_id}`
- 前端调用路径：`/teaching-office/attachments/{evaluationId}` (baseURL: `/api`)
- 实际完整路径：`/api/teaching-office/attachments/{evaluationId}` ✅ 正确

### 2. 权限检查 ✅
后端API权限设置：
```python
# 上传附件 - 需要教研室角色
@router.post("/attachments")
async def upload_attachments(
    current_user: User = Depends(require_teaching_office),
)

# 查询附件 - 任何登录用户都可以
@router.get("/attachments/{evaluation_id}")
def get_attachments(
    current_user: User = Depends(get_current_user),
)

# 下载附件 - 任何登录用户都可以
@router.get("/attachments/{attachment_id}/download")
async def download_attachment(
    current_user: User = Depends(get_current_user),
)
```
权限设置正确 ✅

### 3. 前端代码检查

#### ManualScoringForm.vue 中的附件加载
```typescript
// Load attachments
const loadAttachments = async () => {
  attachmentsLoading.value = true
  try {
    const endpoint = `/teaching-office/attachments/${props.evaluationId}`
    const response = await apiClient.get(endpoint)
    attachments.value = response.data
  } catch (error: any) {
    console.error('Failed to load attachments:', error)
    ElMessage.error('加载附件列表失败')
  } finally {
    attachmentsLoading.value = false
  }
}
```

#### 附件显示逻辑
```typescript
// Get attachments for a specific indicator
const getAttachmentsForIndicator = (indicatorKey: string | number): any[] => {
  const ik = String(indicatorKey)
  const indicatorMap: Record<string, string> = {
    'teachingProcessManagement': 'teachingProcessManagement',
    'teachingQualityManagement': 'teachingQualityManagement',
    // ... 其他映射
  }
  
  const mappedIndicator = indicatorMap[ik] || ik
  return attachments.value.filter(att => att.indicator === mappedIndicator)
}
```

### 4. 可能的问题点

#### 问题1：后端服务未运行 ⚠️
- 后端服务需要在 http://localhost:8000 运行
- 检查方法：访问 http://localhost:8000/api/docs

#### 问题2：自评表ID不匹配 ⚠️
- 前端传递的 `evaluationId` 必须与数据库中的自评表ID一致
- 附件记录的 `evaluation_id` 必须与自评表ID匹配

#### 问题3：附件indicator字段不匹配 ⚠️
- 上传时使用的 `indicator` 值
- 前端查询时使用的 `indicatorKey` 映射
- 两者必须完全一致

#### 问题4：浏览器缓存 ⚠️
- 旧的API响应可能被缓存
- 需要清除浏览器缓存

## 常见错误及解决方案

### 错误1：加载附件列表失败
**症状**：控制台显示"加载附件列表失败"

**可能原因**：
1. 后端服务未运行
2. 自评表ID不存在
3. 网络请求被拦截

**解决方案**：
1. 确认后端服务运行在 http://localhost:8000
2. 检查evaluationId是否正确
3. 查看Network标签的具体错误信息

### 错误2：附件不显示
**症状**：API请求成功，但页面上看不到附件

**可能原因**：
1. indicator字段不匹配
2. attachments数组为空
3. 过滤逻辑有问题

**解决方案**：
1. 在控制台打印 `attachments.value` 查看数据
2. 检查 `getAttachmentsForIndicator` 的返回值
3. 确认indicator映射是否正确

### 错误3：文件下载失败
**症状**：点击下载按钮无反应或报错

**可能原因**：
1. 附件ID不正确
2. 文件在MinIO中不存在
3. 权限不足

**解决方案**：
1. 检查attachment.id是否正确
2. 查看后端日志
3. 确认用户已登录且有权限

### 错误4：401 Unauthorized
**症状**：API请求返回401状态码

**解决方案**：
1. 重新登录
2. 检查token是否过期
3. 清除localStorage并重新登录
