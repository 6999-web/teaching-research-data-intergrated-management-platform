# 重启后端服务

## 需要重启后端服务以应用文件上传修复

### 步骤：

1. **关闭当前后端服务**
   - 找到运行后端的命令行窗口
   - 按 `Ctrl + C` 停止服务

2. **重新启动后端服务**
   ```bash
   cd backend
   python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

3. **验证服务启动**
   - 查看控制台输出
   - 应该看到：`⚠️  MinIO 服务不可用，切换到本地文件存储`
   - 或者：`✅ MinIO 服务连接成功`

4. **测试文件上传**
   - 访问 http://localhost:3000
   - 登录教研室端账号
   - 尝试上传文件

---

## 或者使用快速重启脚本

### Windows:
```bash
# 在项目根目录执行
.\start-local-services.bat
```

这会自动重启前端和后端服务。

---

## 验证修复

重启后，后端会自动：
1. 尝试连接 MinIO
2. 如果连接失败，切换到本地文件存储
3. 在控制台显示当前使用的存储方式

现在文件上传功能应该可以正常工作了！
