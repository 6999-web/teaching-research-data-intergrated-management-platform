# 🔄 清除缓存并重新加载

**日期**: 2026-02-28

---

## ⚠️ 重要提示

如果你看到的页面和最新代码不一致（例如还能看到底部的"拖拽或点击上传"区域），这是因为浏览器缓存了旧版本的页面。

---

## 🚀 快速解决方法

### 方法 1: 硬刷新（推荐）

**Windows / Linux**:
```
按住 Ctrl + Shift + R
或
按住 Ctrl + F5
```

**Mac**:
```
按住 Cmd + Shift + R
```

### 方法 2: 清空缓存并硬性重新加载

1. 按 `F12` 打开开发者工具
2. 右键点击浏览器地址栏旁边的刷新按钮
3. 选择"清空缓存并硬性重新加载"

![清空缓存](https://i.imgur.com/example.png)

### 方法 3: 手动清除缓存

**Chrome / Edge**:
1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存的图片和文件"
3. 时间范围选择"全部时间"
4. 点击"清除数据"

**Firefox**:
1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存"
3. 时间范围选择"全部"
4. 点击"立即清除"

### 方法 4: 无痕模式测试

1. 按 `Ctrl + Shift + N` (Chrome/Edge) 或 `Ctrl + Shift + P` (Firefox)
2. 在无痕窗口中访问 http://localhost:3000
3. 登录并测试功能

---

## ✅ 验证是否成功

清除缓存后，你应该看到：

### ❌ 不应该看到的（旧版本）
- 页面底部有"附件上传（可选）"区域
- 有"选择考核指标"下拉框
- 有"将文件拖到此处，或点击选择"的上传区域

### ✅ 应该看到的（新版本）
- 每个字段右侧有独立的"📎 附件上传（可选）"卡片
- 页面底部只有"提交表单"按钮
- 没有底部的统一附件上传区域

---

## 🎯 完整的刷新步骤

### 步骤 1: 停止前端服务（可选）
如果硬刷新不起作用，可以重启前端服务：

在 Kiro 中：
```
停止进程 ID: 1
然后重新启动前端
```

### 步骤 2: 清除浏览器缓存
按照上面的方法清除缓存

### 步骤 3: 重新访问
1. 关闭所有浏览器标签页
2. 重新打开浏览器
3. 访问 http://localhost:3000
4. 登录并测试

---

## 🔍 检查是否加载了最新代码

### 方法 1: 查看页面源代码
1. 按 `F12` 打开开发者工具
2. 切换到 "Elements" 或 "元素" 标签页
3. 搜索 "附件上传（可选）"
4. 应该只在字段右侧的卡片中找到，不应该在页面底部找到

### 方法 2: 查看 Network 请求
1. 按 `F12` 打开开发者工具
2. 切换到 "Network" 或 "网络" 标签页
3. 刷新页面
4. 查看 `NewSelfEvaluationForm.vue` 的请求
5. 确认状态码是 200，不是 304（304 表示使用了缓存）

## 🎉 提交功能说明

点击"提交表单"按钮后，系统会自动：

1. ✅ 验证表单数据
2. ✅ 保存自评表到数据库
3. ✅ 自动上传所有字段右侧的附件（如果有）
4. ✅ 提交到考评小组
5. ✅ 显示成功提示

**你不需要做任何额外的操作，所有附件会自动随表单一起提交！**

## 📞 仍然有问题？

如果清除缓存后仍然看到旧版本的页面，请：

1. 完全关闭浏览器（所有窗口）
2. 重新打开浏览器
3. 访问 http://localhost:3000

或者尝试使用不同的浏览器测试。
